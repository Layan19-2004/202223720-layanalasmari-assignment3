# AI Usage Report

## Overview

AI tools were used during the development of this project to support coding, debugging, design improvements, and documentation. The purpose of using AI was to improve efficiency, learn better development practices, and solve technical issues while still maintaining full understanding and control of the final project.

All AI suggestions were reviewed, tested, edited, and improved manually before being included in the final submission.

---

## Tools Used and Use Cases

### ChatGPT

ChatGPT was the main AI tool used during this assignment.

Used for:

- Debugging JavaScript issues
- Improving CSS styling and layout
- Generating ideas for Assignment 3 features
- Improving dark mode implementation
- Creating cleaner button behavior
- Helping structure README documentation
- Helping write technical documentation
- Explaining JavaScript logic such as filtering, sorting, and localStorage

---

## How AI Helped This Project

AI support was useful in several areas.

### 1. Problem Solving

When JavaScript logic became complex, AI helped break features into steps.

Examples:

- Combining filtering and sorting together
- Updating dark mode button text dynamically
- Saving user preferences using localStorage
- Handling GitHub API loading and errors

### 2. Code Improvement

AI suggested cleaner ways to organize:

- CSS button styles
- Reusable classes
- Function structure in JavaScript
- Better naming for variables

### 3. Documentation Support

AI helped organize content for:

- README.md
- Technical documentation
- Clear feature explanations

## Benefits of Using AI

Using AI provided several advantages:

- Faster troubleshooting
- Better understanding of JavaScript logic
- Cleaner CSS structure
- Faster idea generation
- More professional documentation
- Improved confidence during development

AI was especially useful when comparing multiple solutions quickly.

---

## Risks, Mistakes, and Limitations of AI

AI was helpful, but not every suggestion was correct or ideal.

### 1. Incorrect or Unnecessary Code

Some generated code was more complex than needed.

Example:

- Overly advanced solutions for simple tasks
- Extra code not suitable for beginner portfolio projects

### 2. Styling Conflicts

Some CSS suggestions caused conflicts with existing button or dark mode styles.

### 3. Generic Documentation

Some generated explanations were too general and needed rewriting to match the actual project.

### 4. Need for Human Review

AI cannot fully understand assignment grading expectations unless guided carefully.

Because of this, relying blindly on AI could reduce marks.

---

## How Problems Were Solved

To ensure quality, every AI suggestion was manually checked.

Steps taken:

- Tested all JavaScript features in browser
- Removed unnecessary code
- Simplified overly complex logic
- Adjusted CSS to match project design
- Rewrote documentation in my own project context
- Compared AI suggestions with assignment rubric

Only working and relevant solutions were kept.

---

## Responsible Use and Academic Integrity

AI was used as a support tool, not as a replacement for learning.

I made sure to:

- Understand all code before using it
- Modify suggestions to fit my project
- Test every feature manually
- Rewrite explanations in my own wording
- Keep control over final decisions

No content was copied blindly without review.
